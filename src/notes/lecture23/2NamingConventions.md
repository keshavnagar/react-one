event naming conventions: 
camelCase: event names in jsx should be written in camelCase.
eg: onClick, onChange, onSubmit
prefix with 'on': event handlers should be prefixed with 'on'.
eg: onClick, onMouseEnter.

function naming conventions: 
prefix with 'handle': it is a commmon convention to prefix event handler functions with 'handle' to clearly indicate their purpose. eg handleClick, handleChange, handleSubmit.

Descriptive name: funciton names should be descriptive and reflect what the function does. eg, handleFormSubmit instead of just handleSubmit if its specifically for form submission.
